# BlockService 区块处理流程图

## 整体架构流程

```
┌─────────────────────────────────────────────────────────────────┐
│                      consumer.go 启动                            │
│  group.Add(block.NewBlockService(...))                          │
└──────────────────────────┬──────────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────────────┐
│              ServiceGroup.Start() 调用                          │
│              每个 BlockService 在独立 goroutine 中运行          │
└──────────────────────────┬──────────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────────────┐
│              BlockService.Start()                               │
│              调用 GetBlockFromHttp()                            │
└──────────────────────────┬──────────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────────────┐
│         GetBlockFromHttp() - 主循环                             │
│  ┌──────────────────────────────────────────────────────────┐  │
│  │  for {                                                    │  │
│  │    select {                                               │  │
│  │      case slot <- slotChan:  // 从通道接收 slot          │  │
│  │        threading.RunSafe(func() {                        │  │
│  │          ProcessBlock(slot)  // 异步处理区块              │  │
│  │        })                                                 │  │
│  │      case <-ctx.Done():     // 服务停止                  │  │
│  │        return                                             │  │
│  │    }                                                      │  │
│  │  }                                                        │  │
│  └──────────────────────────────────────────────────────────┘  │
└──────────────────────────┬──────────────────────────────────────┘
                           │
                           ▼
```

## ProcessBlock 详细流程

```
┌─────────────────────────────────────────────────────────────────┐
│              ProcessBlock(slot) - 处理单个区块                   │
└──────────────────────────┬──────────────────────────────────────┘
                           │
                           ▼
                    ┌──────────────┐
                    │ 参数校验      │
                    │ slot > 0?    │
                    └──────┬───────┘
                           │
                           ▼
                    ┌──────────────┐
                    │ Step0: 初始化 │
                    │ - 创建 Block  │
                    │ - 状态=失败   │
                    └──────┬───────┘
                           │
                           ▼
        ┌──────────────────────────────────────┐
        │  GetSolBlockInfoDelay()              │
        │  通过 RPC 获取区块详细信息            │
        └──────────┬───────────────────────────┘
                   │
        ┌──────────┴──────────┐
        │                     │
        ▼                     ▼
   ┌─────────┐         ┌─────────────┐
   │ 成功    │         │ 失败/跳过   │
   └────┬────┘         └──────┬──────┘
        │                     │
        │                     ▼
        │              ┌─────────────┐
        │              │ 标记状态    │
        │              │ 保存到DB    │
        │              │ return      │
        │              └─────────────┘
        │
        ▼
┌───────────────────────────────────────┐
│ Step1: 同步基础信息                   │
│ - BlockTime (区块时间)                │
│ - BlockHeight (区块高度)              │
│ - Status = Processed                  │
└───────────┬───────────────────────────┘
            │
            ▼
┌───────────────────────────────────────┐
│ Step2: 计算 SOL 价格                  │
│ - 分析区块中的 SOL-USDT/USDC 转账     │
│ - 计算: (USDT|USDC) / SOL = 价格      │
│ - 填充 tokenAccountMap                │
└───────────┬───────────────────────────┘
            │
            ▼
┌───────────────────────────────────────┐
│ Step3: 解码所有交易                   │
│ ┌─────────────────────────────────┐  │
│ │ for each transaction in block:  │  │
│ │   1. DecodeTx()                 │  │
│ │      └─> DecodeInstruction()    │  │
│ │         ├─> PumpFun             │  │
│ │         ├─> PumpFunAMM          │  │
│ │         ├─> TokenProgram        │  │
│ │         └─> Token2022Program    │  │
│ │   2. FillTradeWithPairInfo()    │  │
│ │   3. 添加到 trades 列表          │  │
│ └─────────────────────────────────┘  │
└───────────┬───────────────────────────┘
            │
            ▼
┌───────────────────────────────────────┐
│ Step4: 数据整理                       │
│ - 按交易对(Pair)分组成交              │
│ - 统计不同 Swap 的成交数量            │
└───────────┬───────────────────────────┘
            │
            ▼
┌───────────────────────────────────────┐
│ 处理特殊事件                           │
│ - UpdateTokenMints()  // Mint 事件    │
│ - UpdateTokenBurns()  // Burn 事件    │
└───────────┬───────────────────────────┘
            │
            ▼
┌───────────────────────────────────────┐
│ Step5 & Step6: 并发保存数据            │
│ ┌─────────────────────────────────┐  │
│ │ Goroutine 1:                     │  │
│ │   - SaveTrades()                 │  │
│ │   - SaveTokenAccounts()          │  │
│ └─────────────────────────────────┘  │
│ ┌─────────────────────────────────┐  │
│ │ Goroutine 2:                     │  │
│ │   - SavePumpSwapPoolInfo()       │  │
│ └─────────────────────────────────┘  │
│ Wait for all goroutines               │
└───────────┬───────────────────────────┘
            │
            ▼
┌───────────────────────────────────────┐
│ Step7: 保存区块记录                    │
│ - BlockModel.Insert()                 │
│ - Status = Processed                  │
└───────────────────────────────────────┘
```

## 交易解码流程 (DecodeTx)

```
┌───────────────────────────────────────┐
│         DecodeTx(tx)                  │
└───────────┬───────────────────────────┘
            │
            ▼
    ┌───────────────┐
    │ 参数校验       │
    │ - tx != nil   │
    │ - BlockDb != nil│
    └───────┬───────┘
            │
            ▼
    ┌───────────────┐
    │ 计算交易哈希   │
    │ base58(signature[0])│
    └───────┬───────┘
            │
            ▼
    ┌───────────────┐
    │ 检查交易状态   │
    │ Meta.Err == nil?│
    └───────┬───────┘
            │
            ▼
    ┌───────────────┐
    │ 构建内部指令映射│
    │ GetInnerInstructionMap()│
    └───────┬───────┘
            │
            ▼
    ┌──────────────────────────────┐
    │ for each instruction:        │
    │   DecodeInstruction()        │
```

## 关键组件说明

### 1. BlockService
- **作用**: 区块处理服务的核心结构
- **职责**: 
  - 从通道接收 slot
  - 调用 RPC 获取区块数据
  - 协调整个处理流程

### 2. ProcessBlock
- **作用**: 处理单个区块的主函数
- **步骤**: 
  1. 获取区块信息
  2. 计算 SOL 价格
  3. 解码所有交易
  4. 保存数据

### 3. DecodeTx
- **作用**: 解码单个交易
- **输出**: 成交记录列表

### 4. DecodeInstruction
- **作用**: 根据程序类型解码指令
- **支持的程序**:
  - PumpFun
  - PumpFunAMM
  - TokenProgram
  - Token2022Program

### 5. FillTokenAccountMap
- **作用**: 分析 Token 账户余额变化
- **用途**: 
  - 计算价格
  - 追踪持仓
  - 识别转账操作

## 并发处理

```
主循环 (GetBlockFromHttp)
    │
    ├─> Goroutine 1: ProcessBlock(slot1)
    │       │
    │       └─> Step5 & Step6 并发保存
    │           ├─> Goroutine A: SaveTrades + SaveTokenAccounts
    │           └─> Goroutine B: SavePumpSwapPoolInfo
    │
    ├─> Goroutine 2: ProcessBlock(slot2)
    │       │
    │       └─> Step5 & Step6 并发保存
    │
    └─> Goroutine 3: ProcessBlock(slot3)
            │
            └─> Step5 & Step6 并发保存
```

## 错误处理

```
获取区块失败
    │
    ├─> 错误包含 "was skipped"
    │   └─> 标记为 BlockSkipped，保存到 DB
    │
    └─> 其他错误
        └─> 标记为 BlockFailed，保存到 DB

解码交易失败
    │
    ├─> "unknow program" 错误
    │   └─> 忽略，继续处理下一个交易
    │
    └─> 其他错误
        └─> 记录日志，继续处理下一个交易
```

## 性能优化点

1. **并发处理**: 每个区块在独立 goroutine 中处理
2. **批量保存**: 按交易对分组，批量写入数据库
3. **延迟获取**: GetSolBlockInfoDelay 延迟 1 秒，避免 RPC 限流
4. **预分配**: trades 切片预分配容量 1000
5. **并发保存**: Step5 和 Step6 并发执行
